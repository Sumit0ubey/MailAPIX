## Reporting a Vulnerability

If you discover a vulnerability, report it privately and responsibly.

### Include in your report

- Clear description of the issue
- Impact and potential risk
- Reproduction steps or proof-of-concept
- Suggested mitigation (if known)

### Contact

- GitHub: https://github.com/Sumit0ubey

Please do not open public issues for security vulnerabilities.

## Response Process

1. **Acknowledge** receipt of report
2. **Validate** issue and assess severity
3. **Remediate** with tested fix
4. **Disclose** resolution and release notes as appropriate

## Operational Security Checklist

- Keep all secrets in environment variables
- Use strong SMTP app passwords or provider tokens
- Restrict CORS origins in production
- Enforce HTTPS end-to-end
- Monitor logs and unusual send-rate patterns
- Rotate sensitive credentials periodically
